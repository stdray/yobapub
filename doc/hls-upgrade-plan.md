# План: обновление hls.js для поддержки 4K/HEVC на современных ТВ

## Проблема

Текущая версия hls.js 0.14.x не поддерживает HEVC (H.265). Контент в 4K на KinoPub почти
наверняка отдаётся в HEVC — при H.264 битрейт 4K был бы неоправданно высоким.
Современные телевизоры (Tizen 3.0+, webOS 4.x+) аппаратно декодируют HEVC без проблем,
но hls.js 0.14 не умеет парсить HEVC-кодеки в TS-сегментах через MSE →
`manifestIncompatibleCodecsError`.

## Решение

Загружать разные версии hls.js в зависимости от платформы:
- **hls.js 0.14.x** — для legacy-устройств (Tizen 2.3, старые webOS, или по выбору пользователя)
- **hls.js 1.5+** — для современных устройств (HEVC/4K через MSE)

Это возможно благодаря текущей архитектуре: hls.js подключается как vendor-скрипт
(`externals: { 'hls.js': 'Hls' }`), весь код обращается к `window.Hls`.

## Настройка «Старый телевизор»

Добавить в страницу настроек пункт **«Старый телевизор»** (клиентская настройка,
не серверная):

- **Назначение**: принудительно использовать hls.js 0.14.x вместо автоопределения.
- **Когда нужна**: если на конкретном устройстве hls.js 1.5+ падает или работает
  нестабильно — пользователь может вручную переключиться на проверенную версию
  без потери работоспособности.
- **Реализация**: `storage.ts` → `isLegacyHls(): boolean`, по умолчанию `false`.
  Когда `true` — загружается `hls-legacy.min.js` и отключаются HEVC-фичи
  (эквивалент текущего поведения).
- **Автоматика**: для известных legacy-платформ (Tizen <3, webOS <4) настройка
  игнорируется — всегда legacy.

## План работ

### Фаза 1 — подготовка (без изменения поведения)

1. **Исследовать API-совместимость hls.js 0.14 vs 1.5+**
   - Составить список используемых API: `Events.*`, `Config`, `levels`, `currentLevel`,
     `recoverMediaError()`, `startLoad()`, `stopLoad()`, `destroy()`
   - Найти breaking changes между версиями
   - Определить, нужен ли абстрактный интерфейс-обёртка или API достаточно совместимы

2. **Подготовить vendor-бандлы**
   - `vendor/hls-legacy.min.js` — текущий 0.14.x (с диагностическими патчами)
   - `vendor/hls-modern.min.js` — hls.js 1.5+ (минифицированный)
   - Обновить `CopyWebpackPlugin` в `webpack.config.js`

3. **Добавить настройку «Старый телевизор»**
   - `storage.ts`: `isLegacyHls()` / `setLegacyHls()`
   - `settings.ts`: новый пункт в клиентских настройках (не серверных)

### Фаза 2 — динамическая загрузка

4. **Реализовать выбор версии hls.js**
   - Убрать `<script src="vendor/hls.min.js">` из `index.html`
   - Добавить динамический загрузчик: определяет платформу →
     выбирает `hls-legacy` или `hls-modern` → загружает `<script>` →
     `onload` запускает приложение
   - Логика выбора:
     - `isLegacyHls() === true` → legacy
     - Tizen < 3 → legacy
     - webOS < 4 → legacy
     - Всё остальное → modern

5. **Адаптировать `patch-hls.js`**
   - Текущий патч работает только с 0.14. Для 1.5+ либо сделать отдельный патч,
     либо убрать (в новых версиях seekToStartPos уже управляется лучше)

6. **Обновить типы**
   - Текущие `@types/hls.js@0.13.3` не покрывают 1.5+. hls.js 1.5+ поставляется
     с собственными типами. Решение: использовать пересечение типов или minimal
     interface для общего API

### Фаза 3 — HEVC/4K

7. **Включить серверные флаги на современных ТВ**
   - При загрузке настроек (`/v1/device/settings`) проверить, что `supportHevc`,
     `supportHdr`, `support4k` включены (или подсказать пользователю включить)

8. **Проверить hls4 стриминг с HEVC**
   - На Tizen 3.0+ с hls.js 1.5+ выбрать 4K-контент
   - Убедиться, что hls.js парсит HEVC-манифест, pinQualityLevel работает,
     видео декодируется

9. **Обработка fallback**
   - Если hls.js 1.5+ не может воспроизвести HEVC-стрим (устройство рапортует MSE
     HEVC, но реально не тянет) — показать понятную ошибку с рекомендацией
     снизить качество или включить «Старый телевизор»

### Фаза 4 — тестирование

10. **Тест-матрица**
    - Tizen 2.3 (2015) + hls.js 0.14 → текущее поведение, регрессий нет
    - Tizen 3.0 (2017) + hls.js 1.5+ → 1080p H.264, проверить стабильность
    - Tizen 5.5+ (2020+) + hls.js 1.5+ → 4K HEVC, основной сценарий
    - Android TV (WebView) + hls.js 1.5+ → 4K HEVC
    - Настройка «Старый телевизор» → принудительно 0.14 на любом устройстве
    - webOS (при наличии устройства)

## Риски

- **hls.js 1.5+ ES-уровень**: собирается в ES2015+. На Chromium 47+ (Tizen 3.0+) ES2015
  поддерживается нативно. На Chromium 38 (webOS 3.x) — нет, поэтому legacy.
- **Размер бандла**: hls.js 1.5 ~300KB vs 0.14 ~200KB. Некритично — грузится один раз.
- **MSE HEVC**: не на всех устройствах с аппаратным HEVC-декодером MSE-путь работает.
  Fallback на H.264 quality или на 0.14 через «Старый телевизор».
- **Двойное обслуживание**: два vendor-файла, возможно два patch-скрипта. Приемлемо,
  пока не откажемся от Tizen 2.3.
